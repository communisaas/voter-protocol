# Address-index sample (SEAM-CONTRACT v1 §6)

Real DE/RI/DC index built from real downloaded sources, mirroring the
production layout byte-for-byte:

```
sample/address-index/v1/
├── assertions.json                  # published COPY of the committed corpus (§6 gate input)
├── address-index-sources.json       # download log with computed sha256s
└── US/
    ├── manifest.json                # addressIndex section + third clock
    └── addresses/
        ├── {zip5}.json              # §2 chunk files (299 chunks)
        ├── normalization.json       # §3 Pub 28 tables (normVersion 1)
        └── chunk-index.json         # per-chunk sha256/bytes (§4)
```

## Assertions corpus — committed canon, not a build product

`assertions.json` is NOT regenerated by `build-address-index.ts`. Its
canonical copy is COMMITTED at
`src/__tests__/fixtures/address-index-assertions.json` (27 hand-verified real
addresses incl. 2 MUST-MISS); the file here is a byte-for-byte copy published
alongside the chunks. The corpus is re-derivable and falsifiable on any
machine via an independent §2/§3 re-implementation:

```bash
cd packages/shadow-atlas
npx tsx scripts/verify-address-assertions.ts sample/address-index/v1   # must be 27/27
```

The verifier also fails if this published copy drifts from the committed
canon. When a source-vintage rebuild legitimately moves coordinates:
hand-re-verify the moved entries (`--print-derived` shows the derived
values), update the COMMITTED corpus first, then copy it back here:

```bash
cp src/__tests__/fixtures/address-index-assertions.json sample/address-index/v1/assertions.json
```

Source mix (stamped in `US/manifest.json → addressIndex.sourceMix`): DC
exercises `src:0` (NAD points, `NAD_DC.txt` vintage 2026-06-30) and `src:1`;
DE and RI are `src:1` only (TIGER2025 ADDRFEAT ranges). Both source classes
are exercised, as §6 requires.

## Rebuilding the sample

```bash
cd packages/shadow-atlas
npx tsx scripts/fetch-nad-state.ts --state DC --output data/nad-cache/NAD_DC.txt
npx tsx scripts/build-address-index.ts \
  --nad data/nad-cache/NAD_DC.txt --nad-vintage 2026-06-30 \
  --addrfeat-dir data/addrfeat-cache --addrfeat-vintage TIGER2025 \
  --states DE,RI,DC \
  --output sample/address-index/v1

# The build writes chunks/manifest/sources ONLY — restore the published
# assertions copy from the committed canon, then re-run the acceptance gate:
cp src/__tests__/fixtures/address-index-assertions.json sample/address-index/v1/assertions.json
npx tsx scripts/verify-address-assertions.ts sample/address-index/v1        # 27/27 required
npx tsx scripts/validate-build.ts sample/address-index/v1                   # Address Index check must PASS
#   (boundary checks FAIL on this address-only tree — expected; only the
#    [US] Address Index line gates the sample)
```

## Operator: uploading the sample to R2

The P9 consumer gate fetches this tree at
`https://atlas.commons.email/sample/address-index/v1/US/…`. Upload with the
same script the quarterly chain uses (S3-compatible; requires
`CLOUDFLARE_ACCOUNT_ID`, `R2_ACCESS_KEY_ID`, `R2_SECRET_ACCESS_KEY`):

```bash
cd packages/shadow-atlas
npx tsx scripts/upload-to-r2.ts \
  --directory sample \
  --prefix sample \
  --output sample-upload-results.json
```

`--directory sample --prefix sample` yields object keys
`sample/address-index/v1/US/manifest.json`, `sample/address-index/v1/US/addresses/{zip5}.json`,
`sample/address-index/v1/assertions.json`, … — exactly the §6 layout.

**Known caveat (expected, not a failure of the upload):** every object is
uploaded and byte-verified through the S3 API, but the script's final
public-URL probe checks the quarterly layout (`<prefix>/US/manifest.json` and
`<prefix>/US/district-index.json`), which does not exist one level under
`sample/` — so the run ends with a verification FAILURE exit after the
upload has fully succeeded. Confirm the sample manually instead:

```bash
curl -sf https://atlas.commons.email/sample/address-index/v1/US/manifest.json \
  | jq '.addressIndexGenerated, .addressIndex.totalChunks'
curl -sf https://atlas.commons.email/sample/address-index/v1/assertions.json \
  | jq '.assertions | length'
```

Do NOT re-point the quarterly `upload-r2` job here — production addresses/
travel inside the regular `output/chunked/` tree via the
`chunked-with-addresses` artifact (see `.github/workflows/shadow-atlas-quarterly.yml`,
`build-address-index` job).
